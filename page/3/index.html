对于HTTP的粗浅认识：

打开新世界大门的一天！

菜鸟新手上路，如有错误希望各位大佬指正！本文基于各种网络途径获取的文本资料。
首先是一些词汇的理解问题：

  1. 封装：在读到“最初的主要是用于传递通过HTML封装过的数据。”这句时对于封装一词不解。封装是指隐藏对象的属性和实现细节，仅对外提供公共访问方式,不让外部直接访问实例域，即private,使用者不必关心数据是算法如何实现的，只需要调用数据的方法即可。提高安全性，重用性，更方便使用，将变化隔离。就像机密器械，作为使用者，你可能知道怎么使用，但你不懂是怎么拼装的。
  2. 返回值：在c语言学习时对于返回值不太理解。一个函数的函数名即是函数的代表，也是一个变量，由于函数名变量通常用来把函数的结果带给调函数，即递归调用。所以一般把函数名变量称为返回值（return）。函数可有函数值也可没有函数值，对于没有返回值的函数，功能只是完成一个操作，应将返回值类型定义为void，函数语句体内可无return语句，return 后面不能带任何数据，直接写分号即可。当需要在程序指定位置退出时，可放置一个。
    有强行结束或提前结束函数执行的作用。在做课内练习时，遇到void summary()
    {math.sum=math.score1+math.score2+math.score3;}时对于void的使用存在疑惑，现在明白是因为该函数没有返回值，是在后面的summary();//调用前文的summary中用到。
  3. 主机：主机是指计算机除去输入输出设备以外的主要机体部分。也是用于放置主板及其他主要部件的控制箱体。通常包括 CPU、内存、硬盘、光驱、电源、以及其他输入输出控制器和接口。在网络技术中是关于发送与接收信息的终端设备。我此时重点关注internet主机，与internet相连的任何一名计算机都称为主机，每台主机都有一个唯一的ip地址，在互联网上地位平等。
  4. 端口：一个参数，表示这个数据包到底供哪个程序使用。相当于在相应快递区域的具体某户人。

然后进入对于HTTP的学习。
HTTP(hypertext transfer protocol),超文本传输协议，是互联网上的一个重要协议，最初是主要用于传递通过HTML封装的数据。于此相关的还有FTP文件传输协议等。
对于互联网协议，就不得不提RFC,RFC有所有关于internet的正式标准（request for comment）,为互联网提供了很好的规范。从实体层开始，为互联网运行提供物质基础，可以用光缆、电缆、双绞线、无线电波等方式把电脑连起来，负责传送0和1的电信号。之后便是链接层，规定了0和1的分组方式，之后一种名为“以太网”协议的分组方式占据主导。以太网规定，一组电信号构成一个数据包，叫做”帧”（Frame）。每一帧分成两个部分：标头（Head）和数据（Data），并规定数据包必须是从一块网卡传送到另一块网卡的，网卡的地址叫做MAC地址。而之后的网络层引进了一套新的地址，能够使我们区分不同的计算机是否属于同一个子网络，或是进行路由选择，建立了主机到主机的通信。然后便是重中之重的传输层，自此建立端口到端口的通信，是数据包能够到达相应的客户端，传输需要协议处理，如TCP与UDP协议，这两者都能实现通信但有所不同，TCP可靠，但效率低，而UDP不可靠，但效率高。对于超文本传输HTTP,是用TCP协议。最后是高层：应用层，表示层，会话层；负责主机之间的数据连接。应用层直接服务于客户，提供具体应用，有具体如HTTP协议，FTP协议，DNS协议等具体应用协议。应用层为客户提供的服务和协议有：文件服务、目录服务、文件传输服务（FTP）、远程登录服务（Telnet）、电子邮件服务（E-mail）、打印服务、安全服务、网络管理服务、数据库服务等。上述的各种网络服务由该层的不同应用协议和程序完成，不同的网络操作系统之间在功能、界面、实现技术、对硬件的支持、安全可靠性以及具有的各种应用程序接口等各个方面的差异是很大的。而http作为其中一类应用协议，也在传输过程中控制代理，网关，通道，缓存等。且有两种比较重要的机制，一是连接机制，HTTP/1.1支持两种连接机制，一种是非持久连接，第二种是持久连接。基本上默认是使用持久连接，因为这样能够减少建立连接时候的网络延时与CPU消耗；二是缓存机制， HTTP中使用缓存主要有两个作用，一个是在很多情况下可以减少发送包，减少网络IO，使用“过期”机制来处理；第二个是可以减少发送整包的操作，减少带宽需求，使用“验证”机制来处理。
总的来说，HTTP作为一种得到众多认可的协议，规范了数据包的通信连接，使之井然有序。
